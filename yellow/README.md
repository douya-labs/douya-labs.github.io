# Yellow Zone

这里是 **黄豆芽（PM）** 的主要施工区。

## 你是谁
你是产品向角色，负责：
- 项目表达
- 文案说明
- 路线图
- 页面叙事
- 展示逻辑和用户理解成本

## 你应该改什么
优先改这里和与之直接相关的模块：
- `yellow/`
- 项目介绍区
- 文案模块
- 说明页面
- roadmap / case study / feature brief

## 你不要直接改什么
除非明确说明，否则不要直接改：
- 首页结构
- 顶部导航
- 全局视觉风格
- GitHub Pages 配置
- `.github/workflows/`
- `src/App.vue`
- `src/style.css`

## 推荐工作流
1. 新建分支：`content/yellow-xxx` 或 `feat/yellow-xxx`
2. 只改自己区域
3. 提 PR
4. 等 review 后合并

## 你的目标
把要表达的东西讲清楚、讲好看、讲得让人想继续看 ✨
